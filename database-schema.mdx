# Database Schema — GMC Education Portal

**Document ID:** DB-SCHEMA-01  
**Database:** PostgreSQL 15+  
**Last Updated:** 2026  
**Owner:** Tech Lead  
**Encoding:** UTF8 (utf8mb4 equivalent — supports Gujarati, Hindi, English)

---

## Architecture Overview

```
municipalities
    └── schools
            ├── academic_years
            ├── classes
            │     ├── students
            │     ├── timetable_slots
            │     └── class_subjects (class ↔ subject ↔ teacher mapping)
            ├── teachers
            ├── subjects
            └── parents (linked via student_parents)

Per Class + Subject:
    ├── materials (study content uploaded by teacher)
    ├── assignments
    │     └── submissions (student responses)
    ├── attendance_records
    │     └── attendance_entries (per student per day)
    └── marks
          └── mark_entries (per student per assessment)

System:
    ├── users (auth table — all roles)
    ├── roles + permissions
    ├── notifications
    ├── offline_sync_queue
    └── audit_logs
```

## 4. Timetable

### timetable_slots

```sql
CREATE TABLE timetable_slots (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  class_id         UUID NOT NULL REFERENCES classes(id),
  class_subject_id UUID NOT NULL REFERENCES class_subjects(id),
  school_id        UUID NOT NULL REFERENCES schools(id),
  day_of_week      INTEGER NOT NULL CHECK (day_of_week BETWEEN 1 AND 6), -- 1=Mon, 6=Sat
  period_number    INTEGER NOT NULL,
  start_time       TIME NOT NULL,
  end_time         TIME NOT NULL,
  effective_from   DATE NOT NULL,
  effective_until  DATE,
  created_at       TIMESTAMPTZ DEFAULT NOW(),

  -- No two slots for same class at same time
  UNIQUE (class_id, day_of_week, period_number, effective_from)
);

-- Ensure a teacher is not double-booked across different classes in the same slot
CREATE OR REPLACE FUNCTION check_teacher_double_booking()
RETURNS TRIGGER AS $$
DECLARE
  new_teacher_id UUID;
  conflict_found BOOLEAN;
BEGIN
  -- Get teacher_id for the slot being scheduled
  SELECT teacher_id INTO new_teacher_id
  FROM class_subjects
  WHERE id = NEW.class_subject_id;

  -- Check if this teacher is already booked in another class at the same slot
  SELECT EXISTS (
    SELECT 1
    FROM timetable_slots ts
    JOIN class_subjects cs ON ts.class_subject_id = cs.id
    WHERE cs.teacher_id = new_teacher_id
      AND ts.day_of_week = NEW.day_of_week
      AND ts.period_number = NEW.period_number
      AND ts.id != NEW.id -- ignore self on updates
      AND (ts.effective_until IS NULL OR NEW.effective_from <= ts.effective_until)
      AND (NEW.effective_until IS NULL OR ts.effective_from <= NEW.effective_until)
  ) INTO conflict_found;

  IF conflict_found THEN
    RAISE EXCEPTION 'Teacher % is already booked in another class for day %, period %',
      new_teacher_id, NEW.day_of_week, NEW.period_number;
  END IF;

  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER enforce_no_teacher_double_booking
BEFORE INSERT OR UPDATE ON timetable_slots
FOR EACH ROW EXECUTE FUNCTION check_teacher_double_booking();
```

### student_attendance_summary

```sql
CREATE MATERIALIZED VIEW student_attendance_summary AS
SELECT
  sc.student_id,
  sc.class_id,
  sc.school_id,
  sc.academic_year_id,
  COUNT(ae.id) FILTER (WHERE ae.status = 'PRESENT') AS present_days,
  COUNT(ae.id) FILTER (WHERE ae.status != 'HOLIDAY') AS total_days,
  ROUND(
    COUNT(ae.id) FILTER (WHERE ae.status = 'PRESENT') * 100.0
    / NULLIF(COUNT(ae.id) FILTER (WHERE ae.status != 'HOLIDAY'), 0),
    2
  ) AS attendance_percentage
FROM student_classes sc
LEFT JOIN attendance_entries ae ON ae.student_id = sc.student_id
LEFT JOIN attendance_records ar ON ar.id = ae.attendance_record_id
  AND ar.class_id = sc.class_id
GROUP BY sc.student_id, sc.class_id, sc.school_id, sc.academic_year_id;

CREATE UNIQUE INDEX ON student_attendance_summary(student_id, academic_year_id);

### Enforce Storage Quotas

```sql
CREATE OR REPLACE FUNCTION check_storage_quota()
RETURNS TRIGGER AS $$
DECLARE
  quota INTEGER;
  used  INTEGER;
BEGIN
  SELECT storage_quota_mb, storage_used_mb
  INTO quota, used
  FROM schools WHERE id = NEW.school_id;

  -- Convert uploaded file size from KB to MB and verify
  IF (used + (NEW.file_size_kb / 1024.0)) > quota THEN
    RAISE EXCEPTION
      'Storage quota exceeded for school. Used: %MB, Quota: %MB',
      used, quota;
  END IF;

  -- Update used storage
  UPDATE schools
  SET storage_used_mb = storage_used_mb + (NEW.file_size_kb / 1024.0)
  WHERE id = NEW.school_id;

  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER enforce_storage_quota
BEFORE INSERT ON materials
FOR EACH ROW EXECUTE FUNCTION check_storage_quota();

-- Enforce assignment submission deadline & grace period at DB level
CREATE OR REPLACE FUNCTION check_submission_deadline()
RETURNS TRIGGER AS $$
DECLARE
  deadline      TIMESTAMPTZ;
  grace_minutes INTEGER;
BEGIN
  SELECT deadline_at, grace_period_min
  INTO deadline, grace_minutes
  FROM assignments
  WHERE id = NEW.assignment_id;

  -- Hard cutoff = deadline + grace period
  IF NEW.submitted_at > (deadline + (grace_minutes || ' minutes')::INTERVAL) THEN
    RAISE EXCEPTION
      'Submission rejected. Deadline was %, grace period % minutes.',
      deadline, grace_minutes;
  END IF;

  -- Mark as late if after deadline but within grace
  IF NEW.submitted_at > deadline THEN
    NEW.is_late := TRUE;
  END IF;

  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER enforce_submission_deadline
BEFORE INSERT ON submissions
FOR EACH ROW EXECUTE FUNCTION check_submission_deadline();

CREATE INDEX idx_mark_entries_assessment ON mark_entries(assessment_id);

-- Enforce marks_obtained <= assessment.max_marks
CREATE OR REPLACE FUNCTION check_marks_limit()
RETURNS TRIGGER AS $$
DECLARE
  max INTEGER;
BEGIN
  SELECT max_marks INTO max
  FROM assessments
  WHERE id = NEW.assessment_id;

  IF NEW.marks_obtained > max THEN
    RAISE EXCEPTION 'Marks % exceed max_marks %',
      NEW.marks_obtained, max;
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER enforce_marks_limit
BEFORE INSERT OR UPDATE ON mark_entries
FOR EACH ROW EXECUTE FUNCTION check_marks_limit();
```

## 12. Row-Level Security (RLS)

PostgreSQL RLS ensures school data isolation at database level — not just application level.

```sql
-- Enable RLS on all sensitive tables
ALTER TABLE students ENABLE ROW LEVEL SECURITY;
ALTER TABLE attendance_records ENABLE ROW LEVEL SECURITY;
ALTER TABLE mark_entries ENABLE ROW LEVEL SECURITY;
ALTER TABLE materials ENABLE ROW LEVEL SECURITY;

-- Example policy: Teacher sees only their school's data
CREATE POLICY school_isolation ON attendance_records
  USING (school_id = current_setting('app.current_school_id')::UUID);

-- Set at start of every request:
SET app.current_school_id = '<school_id_from_jwt>';
```

## 14. Key Design Decisions

**UUIDs everywhere** — not integer IDs. Prevents ID enumeration attacks. Safe to expose in URLs.

**school_id denormalized on most tables** — redundant but critical. Allows RLS policies and fast queries without always joining back to schools table.

**Soft deletes only** — no hard DELETE on academic data. status = ARCHIVED / INACTIVE. Real deletion only on personal data per DPDP erasure request.

**TIMESTAMPTZ not TIMESTAMP** — all timestamps stored with timezone (UTC). Displayed in IST (Asia/Kolkata) in application layer. Prevents the deadline timezone bug.

**JSONB for metadata and audit** — flexible enough for evolving data without schema migrations.

**Audit log is append-only** — DB user has INSERT only on audit_logs. No UPDATE, no DELETE. Even accidental code cannot delete audit history.

**Checksum on all files** — SHA256 stored with every uploaded file. Verified on download. Catches corruption.

**token_version on users** — incrementing this field invalidates all JWTs for that user instantly — used on password change, account suspension.
